# Appendix

## 목적

본문 허브 문서에서 제거한 이력, 상세 수용 기준, 리스크, 용어를 보관한다.

## 문서 맵

- [feature-history](./appendix/feature-history.md)
  - `F01~F38`, `BUG-01~02` 완료 이력
- [detailed-acceptance](./appendix/detailed-acceptance.md)
  - 영역별 상세 수용 기준
- [backlog-and-risks](./appendix/backlog-and-risks.md)
  - Known issue, 범위 밖 항목, 운영 리스크
- [glossary](./appendix/glossary.md)
  - 반복 용어 정의

## 사용 원칙

1. 허브 문서에는 긴 이력표를 두지 않는다.
2. 기능 현황은 `FEATURE_INDEX.md`, 역사적 기록은 `feature-history.md`에 둔다.
3. 새로운 리스크를 추가할 때는 가능하면 대응 도메인/계약 문서와 함께 갱신한다.
